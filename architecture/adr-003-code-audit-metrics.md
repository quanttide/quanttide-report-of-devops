# ADR-003: code audit 指标清单与采纳标准

- **状态**: 已采纳
- **日期**: 2026-07-06
- **涉及项目**: qtcloud-devops

## 背景

`code audit` 从最初的 3 项检查逐步扩充到 6 项。需要明确哪些指标值得加、哪些不该加。

## 当前指标

| 指标 | 测量方法 | 阈值 |
|------|----------|------|
| Scope 目录完整性 | `contract.yaml` 中的 `dir` 路径是否存在 | 全部存在 |
| TODO/FIXME/HACK 密度 | 扫描 `todo`/`fixme`/`hack`，千行密度 | < 5‰ |
| unwrap/expect 密度 | 扫描 `.unwrap()`/`.expect(`，千行密度 | < 10‰ |
| 导入数 | 统计 `use`/`import` 行数 | ≤ 30 个 import |
| 文件长度 | 统计文件行数 | ≤ 500 行 |
| 语法检查 | 按语言执行外部 lint 命令 | 全部通过 |

## 采纳标准

新指标必须满足：

1. **不需要 parser**（不引入 tree-sitter 等 AST 库）
2. **可操作**——看到结果就知道该修什么
3. **不产生噪声**——不报告 codegen 文件等无意义的违规

## 被拒绝的指标

| 指标 | 拒绝原因 |
|------|----------|
| 修改热度（commit 数） | monorepo 中主要是子模块指针变更，不可操作 |
